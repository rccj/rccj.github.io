What is "this" in JavaScript

發表於 2021-11-20   |   更新於 2021-11-20 | 分類於 JavaScript ， 奇怪部分 |

前言

此次筆記根據姚偉揚老師的直播影片[一次搞懂 JavaScript 的 #this、this 還有那個 this

](https://www.facebook.com/hilivecode/videos/190238981712769)做的紀錄，有興趣的可以直接看影片，主講者講得很清楚，還會跟你互動。

this 是什麼

直接印出this是Global物件，瀏覽器裡就是Window
為方便以下統稱Window

在一般函式裡

Q: 請問這個this是什麼?

A: 不知道

就是不知道!
因為…

函式裡的this是取決的函式是怎麼執行的

1. 直接執行函式

this是Window

2. 作為物件的成員函式執行

this是該物件

那這樣呢?

這就會回到第一個分類直接執行函式，this是Window

3. 作為 DOM 事件偵聽函式

this是該 DOM本身

注意

作為非DOM的偵聽，this是不一定

4. 作為建構函式

this是建構出來的實例

bind/apply/call的this

bind

強制把函式裡的this綁訂到目標物上，產生新的函式。

this是 Number{0}

如果是這樣

this是 obj

必須留意的是，被bind綁訂的函式會創造出一個新的函式

a跟b是不同的函式，但他們內容是一樣的

補充

bind也可以綁訂參數

apply

跟bind有點像，但他是綁訂後直接執行

call

跟apply一樣，接受參數方式不同

綁訂箭頭函式

以上執行的都是Window
說明bind/apply/call對箭頭函式是無效的
